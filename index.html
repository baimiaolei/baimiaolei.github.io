react--按需加载

发表于 2018-07-24

我又来啦。
页面加载慢，需要做按需加载，万万没想到，react-router4的按需加载，不能用webpack以往
const index = r => require.ensure([], () => r(require('../components/index')), 'index')
然后，查资料，提供了一个思路，使用bundle,但是出现了一个问题，this.props.history,找不到。。。
后来找到一个方法，使用withRouter,可是我看不懂，也不会用。
算了，不废话了，祭出终级神器，prop-types
这个可以用npm install prop-types 安装.
然后在需要使用this.props.history的界面，在constructor上面这样写：
static contextTypes = { router: PropTypes.object.isRequired };
然后使用 this.context.router.history 就可以跳转了 ^_^ ~

有关type=file的问题

发表于 2018-07-19

<input type="file" id="file" />
1.鼠标一直不能变成pointer ,百度之后发现，需要设置font-size:0
2.选择接受文件的类型 添加accept属性 例如：accept=”image/*”，接受所有图片类型的文件
3.多文件上传 添加 multiple=”multiple”

布局：
<div className="uploadWarpper"> <input type="file" id="file" multiple="multiple" accept="image/*" /> <p className="uploadImg"></p> </div>
外层relative定位，input和p标签 absolute 定位，input的z-index比p标签高一点
p标签内部放图片
css:
.uploadWarpper{
width: 200px;
height: 200px;
position: relative;
margin-right:15px;
float: left;
}

#file{
cursor: pointer;
opacity: 0;
position: absolute;
z-index: 2;
width: 200px;
height: 200px;
}
.uploadImg{
cursor: pointer;
position: absolute;
width: 200px;
height: 200px;
z-index: 1;
border:1px dashed #dfdfdf;
}
.uploadImg:after{
content:’+’;
font-size: 60px;
color:#dfdfdf;
display: inline-block;
width: 100%;
line-height: 200px;
text-align: center;
}

hexo--基础使用

发表于 2017-11-24 | 分类于 搭建博客

hexo–图片的引入

上面有一片数组循环的文章，我当时想引入自己作的思维导图，但是不懂markdown语法 ，不知道如何引入。上网搜索一番，总共有两个方法
1.用插件
安装 npm install hexo-asset-image –save
2.不用插件
首先 hexo new “XXX”
使用完命令之后，在source/_post文件夹里面就会出现一个“XXX.md”的文件和一个“XXX”的文件夹。
把需要引入的图片放入文件夹中
然后在页面使用
![](XXX/XXX.png) 就可以了

差点忘了，不论你使用插件还是不使用插件，你都需要修改_config.yml配置文件post_asset_folder项为true。

hexo 如何删除文章

  • 本地删除
  • 使用命令
    hexo g
    hexo d
    就可以了

hexo 发布文章

  • 使用命令
    hexo d -g

hexo 更新文章没变化的时候

  • 手动删除 .deploy_git
  • 执行hexo clean
  • hexo d -g 就可以了
