git 相关问题（持续更新）

发表于 2018-11-06

1.git push 时每次需要输用户名和密码
问题：换了个ssh key,发现每次git push origin master的时候都要输入用户名和密码
原因: 在添加远程库的时候使用了https的方式,所以每次都要用https的方式push到远程库
查看使用的传输协议：
git remote -v

重新设置成ssh的方式：
git remote rm origin git remote add origin git@github.com:username/repository.git git push -u origin master
2.git命令行添加文件夹及目录
git add 文件夹/
git add 文件名.文件类型

  1. git添加远程仓库
    git remote add origin git@github.com:username/repository.git
    4.推送到远程仓库
    第一次推送到远程仓，-u的意思是Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。
    git push -u origin master
    5.将本地文件推送到远程仓库的步骤
    git add 文件名.文件类型
    git commit -m '修改信息'
    git push origin master
